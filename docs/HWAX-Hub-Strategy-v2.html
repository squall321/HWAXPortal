02 각 팀 참여 유형 — 4가지 유형 1·2 = 팀 상황에 따라  ·  유형 3·4 = 전 팀 필수 (Top-down)
해당 팀 · 협의

유형 1 — 팀 포탈 연동
팀 내 자체 포탈 서비스를 운영하는 팀
팀 포탈은 계속 자체 운영 — 단, 통합 포탈과 공통 API 규격(데이터 주고받는 방식)을 협의
혜택: 공통 데이터 수집 API를 제공받아 타 팀 데이터 상호 접근 + 사내 시스템 데이터 활용 가능
팀 포탈 API 개방은 필수 — 통합 포탈이 데이터를 가져가 보관(캐싱)해 데이터 사일로(고립) 방지
연동 담당자 1명 지정 — 규격 협의·지속 관리
💬 데이터를 여는 만큼, 개발실 전체 데이터를 받아 쓸 수 있음
해당 팀 · 신청
⚙️
유형 2 — 서비스 운영 위탁
자동화 프로그램은 개발하지만, 웹 서비스 운영 지원이 필요한 팀
개발·테스트는 기존처럼 자체 수행 — 바뀌는 것 없음
완성된 소스를 정해진 방식으로 GitHub에 올리면, 허브가 자동으로 웹 서비스화
서버·배포·장애 대응은 HE팀이 대행 → 팀은 개발에만 집중
Python, Java, Node.js, C++ 등 15종 이상 개발 언어 지원
📋 등록 방법 안내 후, GitHub 업로드만 하면 자동 운영 시작
전 팀 필수 · Top-down
🤖
유형 3 — 생성형 AI 연계
팀 데이터·챗봇을 생성형 AI 서비스에 탑재
각 팀의 데이터와 챗봇을 생성형 AI 서비스(AX Data Hub)에 탑재
팀 간 데이터가 연결되어 시너지 — 한 곳에서 여러 팀의 데이터를 함께 질의·활용
그룹 데이터 담당자 지정 → 업로드·품질 관리 전담
ChatGPT / Claude 오픈 후 — LLM 활용 데이터 업로드 방법 공유 예정
🔑 개별 팀의 선택이 아닌, 전 팀 공통 추진 사항 (담당님 지시 필요)
