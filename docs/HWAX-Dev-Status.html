05핵심 개념 — 쉽게 풀이이 자료에 나오는 어려운 용어를 "무엇을 하는 것인지"로 풀었습니다
통합 로그인 (SSO)
회사 출입증 하나로 모든 건물을 드나들듯, 로그인 한 번이면 여러 AI 서비스에 다시 로그인 없이 들어갑니다.
MCP (AI 도구 연결 표준)
AI 채팅이 각 시스템의 기능을 직접 불러 쓰게 하는 '표준 콘센트'. "VOC 분석해줘"라고 하면 AI가 분석 기능을 직접 실행합니다.
MCP Gateway (도구 관문)
시스템마다 흩어진 그 콘센트를 하나의 관문으로 모아, 보안·인증·기록을 한곳에서 관리합니다. (다음 단계 과제)
AI 에이전트 (스스로 판단)
AI가 질문을 보고 '어떤 도구를 써야 하나' 스스로 골라 실행하고 답하는 똑똑한 비서.
토큰 · PAT (프로그램용 출입증)
사람의 비밀번호 대신 프로그램이 쓰는 장기 출입증. 새어 나가도 그것만 폐기하면 되어 안전합니다.
시맨틱 검색 · RAG (뜻으로 찾고 근거로 답함)
단어가 달라도 의미가 비슷하면 찾아주고, 찾은 우리 자료를 근거로 AI가 답하게 합니다. 추측이 아닌 근거 기반.
사내 LLM (회사 안 AI 모델)
데이터가 외부로 안 나가게 사내 GPU에서 직접 돌리는 대형 AI 모델. 현재 7B → 추후 72B(고성능).
자동 서비스화 · 사내망 배포
개발자가 소스만 올리면 알아서 웹서비스로 띄워주고, 인터넷 막힌 사내망에서도 설치되게 미리 묶어 보냅니다.
06MCP로 모든 절차를 지식 자산화흩어진 산출물 → MCP 연결 → LLM 증강 → 재사용 자산
핵심은, 모든 업무 절차에 MCP를 끼워 무엇이든 LLM과 연결하고 지식 자산으로 쌓는 흐름이다.
만들면 흩어져 사라지던 산출물(PPT·보고서·LLM 대화·OCR 문서·시뮬레이션·사내 사이트·raw 데이터)이, MCP를 통해 검색·증강·재사용되는 자산으로 전환된다.
1
캡처
PPT·보고서·LLM 대화·OCR 문서·시뮬레이션 결과·사내 사이트·raw 데이터 — 어떤 산출물이든 입력이 된다.
→
2
MCP 등록
각 산출물을 MCP로 시스템(Report Archive · AI Data Hub · MX White Paper)에 등록·연결한다.
→
3
LLM 증강
검색·요약·내용 보강(augment)·백서화·대화 문서화 — LLM이 원자료를 정제된 지식으로 가공한다.
→
4
지식 자산
검색·인용 가능한 지식으로 축적. 부서·조직이 재사용하고, 다시 LLM의 근거가 된다.
선순환 — 쌓인 지식 자산은 다시 MCP로 LLM에 연결되어 다음 작업의 근거가 된다. 쓸수록 자산이 늘고, 늘수록 똑똑해지는 구조.
07지식 자산화 시나리오 — 무엇이든 자산이 된다실제 절차별 흐름 — 입력에서 자산까지
PPT 보강

대화 문서화

데이터 등록

흩어진 사내 AI 서비스를 하나로 묶어, 한 번 로그인하면 모든 곳을 통과하고 채팅으로 데이터를 묻게 하는 허브.
무엇인가
사내 AI 서비스들의 단일 진입 허브. 로그인·접근·채팅을 한 곳에서 처리하는 관문으로, 사용자에게는 '여러 사이트'가 아니라 '하나의 포털'로 보인다.
왜 필요한가
팀마다 서비스·로그인·데이터가 따로라 매번 다시 로그인하고 데이터가 단절됨. 신규 서비스가 생길 때마다 주소·로그인을 또 외워야 하는 부담을 하나의 관문으로 없앤다.
핵심 컨셉
출입증 하나로 모든 건물을 드나들듯 — 한 번 로그인(SSO)하면 재로그인 없이 모든 서비스 진입. 같은 화면 채팅으로 데이터를 묻고, 토큰은 주소창에 안 남게 안전 전달.
페더레이션 역할
모든 사용자의 진입점이자 인증 발급자(IdP 브로커). 5개 서비스가 이 허브 뒤에 붙어 SSO로 연결되고, 신규 서비스는 '복제'만으로 온보딩된다.
한 줄 가치 · 사용자는 한 곳만 기억하면 되고, 서비스들은 사용자·데이터를 공유한다.
